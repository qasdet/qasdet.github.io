Продвинутые техники тестирования на Python

Опубликовано 25 января 2024

Когда вы освоили основы автоматизации тестирования на Python, приходит время изучить более продвинутые техники. Они помогут сделать ваши тесты более надежными, эффективными и поддерживаемыми.

Фикстуры в pytest

Фикстуры — это мощный механизм pytest для подготовки тестового окружения. Они позволяют избежать дублирования кода и управлять ресурсами тестов.

import pytest

@pytest.fixture
def user_data():
    return {
        "name": "Иван Иванов",
        "email": "ivan@example.com",
        "age": 30
    }

def test_user_name(user_data):
    assert user_data["name"] == "Иван Иванов"

def test_user_email(user_data):
    assert "@" in user_data["email"]

Фикстуры могут иметь разную область видимости (function, class, module, session) и могут выполнять код до и после тестов.

Параметризованные тесты

Вместо написания множества похожих тестов, можно использовать параметризацию:

import pytest

@pytest.mark.parametrize("a, b, expected", [
    (2, 3, 5),
    (1, 1, 2),
    (0, 5, 5),
    (-1, 1, 0),
])
def test_addition(a, b, expected):
    assert a + b == expected

Это позволяет запускать один и тот же тест с разными наборами данных, что особенно полезно для проверки граничных условий.

Моки и стабы

Моки позволяют имитировать поведение внешних зависимостей, таких как API или базы данных:

from unittest.mock import Mock, patch
import requests

def get_user_data(user_id):
    response = requests.get(f"https://api.example.com/users/{user_id}")
    return response.json()

@patch('requests.get')
def test_get_user_data(mock_get):
    # Настраиваем мок
    mock_get.return_value.json.return_value = {
        "id": 1,
        "name": "Иван"
    }
    
    # Выполняем тест
    result = get_user_data(1)
    
    # Проверяем результат
    assert result["name"] == "Иван"
    mock_get.assert_called_once_with("https://api.example.com/users/1")

Тестирование веб-приложений с Selenium

Для более сложных сценариев тестирования веб-приложений используйте Page Object Model:

from selenium import webdriver
from selenium.webdriver.common.by import By

class LoginPage:
    def __init__(self, driver):
        self.driver = driver
        
    def enter_username(self, username):
        self.driver.find_element(By.ID, "username").send_keys(username)
        
    def enter_password(self, password):
        self.driver.find_element(By.ID, "password").send_keys(password)
        
    def click_login(self):
        self.driver.find_element(By.ID, "login-button").click()
        
    def login(self, username, password):
        self.enter_username(username)
        self.enter_password(password)
        self.click_login()

# Использование в тесте
def test_successful_login():
    driver = webdriver.Chrome()
    login_page = LoginPage(driver)
    login_page.login("user", "password")
    # Проверяем успешный вход
    assert "dashboard" in driver.current_url
    driver.quit()

Работа с API

Для тестирования REST API используйте библиотеку requests вместе с pytest:

import requests
import pytest

BASE_URL = "https://api.example.com"

def test_create_user():
    user_data = {
        "name": "Новый пользователь",
        "email": "newuser@example.com"
    }
    
    response = requests.post(f"{BASE_URL}/users", json=user_data)
    
    assert response.status_code == 201
    created_user = response.json()
    assert created_user["name"] == user_data["name"]
    assert created_user["email"] == user_data["email"]

def test_get_user():
    response = requests.get(f"{BASE_URL}/users/1")
    
    assert response.status_code == 200
    user = response.json()
    assert "name" in user
    assert "email" in user

Отчеты и интеграция

Для генерации красивых отчетов используйте плагины pytest:

# Установка плагинов
# pip install pytest-html pytest-xdist

# Генерация HTML отчета
# pytest --html=report.html

# Параллельный запуск тестов
# pytest -n 4

Это позволяет запускать тесты параллельно и получать подробные отчеты о результатах.

Лучшие практики

1. Используйте осмысленные имена для тестов. Название теста должно ясно указывать, что именно проверяется.

2. Следуйте принципу AAA. Arrange (подготовка), Act (действие), Assert (проверка).

3. Делайте тесты независимыми. Каждый тест должен работать независимо от других.

4. Используйте фикстуры для подготовки данных. Это упрощает поддержку тестов.

5. Покрывайте граничные условия. Тесты на 0, отрицательные числа, пустые строки и т.д.

Помните главное

Продвинутые техники тестирования — это не про сложность ради сложности, а про эффективность и надежность. Используйте эти подходы для создания тестов, которые легко читаются, поддерживаются и действительно помогают находить ошибки. Помните, что цель автоматизации — не заменить ручное тестирование, а дополнить его и освободить время для более творческой работы.
