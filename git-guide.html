Git Cheat Sheet

Шпаргалка по Git: коммиты, конфликты, история, восстановление и лайфхаки

📝 Коммиты

▶ Изменить последний коммит
Изменить сообщение
git commit --amend -m "Новое сообщение"
Работает только для непушнутых коммитов.
Добавить забытые файлы
git add --all && git commit --amend --no-edit
▶ Отменить коммиты
Отменить (soft) — изменения в индексе
git reset --soft HEAD~1
Отменить (mixed) — изменения не в индексе
git reset HEAD~1
Отменить (hard) — удалить изменения
git reset --hard HEAD~1
ОПАСНО! Необратимо.
Отменить конкретный коммит
git revert <hash>
Создаёт новый коммит с обратными изменениями.
▶ Создание коммита
Коммит с сообщением
git commit -m "feat: add login"
Коммит всех отслеживаемых
git commit -am "update"
Пустой коммит (триггер CI)
git commit --allow-empty -m "trigger build"

📜 История

▶ Просмотр истории
Красивый лог с графом
git log --oneline --graph --all --decorate
Лог конкретного файла
git log --oneline -- <file>
Лог с изменениями строк
git log -p <file>
Кто написал какую строку
git blame <file>
▶ Поиск в истории
Найти по сообщению
git log --oneline --grep="fix"
Найти по автору
git log --oneline --author="name"
Найти коммит с кодом
git log -S "function_name" --oneline
▶ Сравнение
Разница между ветками
git diff main..feature
Изменения файла за 5 коммитов
git diff HEAD~5 -- <file>
Показать содержимое коммита
git show <hash>
Разница с индексом
git diff --cached

⚔️ Конфликты

▶ Прерывание и отмена
Отменить merge
git merge --abort
Отменить rebase
git rebase --abort
Отменить cherry-pick
git cherry-pick --abort
▶ Разрешение конфликтов
Список конфликтующих файлов
git diff --name-only --diff-filter=U
Принять их версию
git checkout --theirs <file>
Принять свою версию
git checkout --ours <file>
Визуальный редактор
git mergetool
▶ После разрешения
Завершить merge
git add <file> && git commit
Завершить rebase
git add <file> && git rebase --continue
Пропустить коммит
git rebase --skip

🌿 Ветки

▶ Основные операции
Список локальных
git branch
Список всех
git branch -a
Создать и переключиться
git checkout -b <name>
Переключиться
git checkout <name>
Удалить локальную
git branch -d <name>
Удалить удалённую
git push origin --delete <name>
▶ Переименование и трекинг
Переименовать текущую
git branch -m <new-name>
Привязать к remote
git branch -u origin/main
Список отслеживаемых
git branch -vv
Предыдущая ветка
git checkout -

📦 Stash

▶ Сохранение
Сохранить всё
git stash
С сообщением
git stash push -m "WIP: login"
Включая неотслеживаемые
git stash -u
Без индексированных
git stash --keep-index
▶ Применение
Применить и удалить
git stash pop
Применить без удаления
git stash apply
Конкретный запись
git stash apply stash@{2}
▶ Управление
Список
git stash list
Посмотреть
git stash show -p stash@{0}
Удалить
git stash drop stash@{0}
Очистить всё
git stash clear
Создать ветку из stash
git stash branch <name> stash@{0}

🔄 Rebase

▶ Интерактивный rebase
Последние N коммитов
git rebase -i HEAD~5
pick — оставить | squash — слить | fixup — слить без msg | reword — изменить msg | drop — удалить
Перенести ветку на main
git rebase main
▶ Продвинутые
Executes during rebase
x exec "npm test"
Взять конкретный коммит
git rebase --onto <new> <old>

🍒 Cherry-pick

▶ Перенос коммитов
Один коммит
git cherry-pick <hash>
Несколько
git cherry-pick <hash1> <hash2>
Диапазон
git cherry-pick <start>..<end>
Без коммита
git cherry-pick -n <hash>

🩹 Восстановление

▶ Reflog — журнал действий
История всех действий
git reflog
Показывает ВСЕ изменения HEAD за 90 дней.
Восстановить после reset
git reflog # найти хеш git reset --hard <hash>
▶ Файлы
Восстановить из коммита
git checkout <hash> -- <file>
Восстановить последний
git restore <file>
Восстановить удалённый
git checkout HEAD~1 -- <file>
▶ Потерянные коммиты
Найти все коммиты
git fsck --lost-found
Найти detached коммиты
git log --all --oneline | grep "detached"

🔒 Секреты и история

▶ Удаление из истории
Удалить файл из всех коммитов
git filter-repo --invert-paths --path=<file> --force
Рекомендуемый способ (нужен filter-repo).
Через filter-branch
git filter-branch --force --index-filter "git rm --cached --ignore-unmatch <file>" --prune-empty --tag-name-filter cat -- --all
▶ GitHub Secret Scanning
Разблокировать секрет
https://github.com/OWNER/REPO/security/secret-scanning/unblock-secret/ID
Профилактика
echo "TOKEN=*" >> .gitignore git update-index --assume-unchanged <file>

🐙 GitHub

▶ Push, Pull, Fetch
Клонировать
git clone URL
Конкретная ветка
git clone -b <branch> URL
Pull с rebase
git pull --rebase origin main
Force push
git push --force
ОПАСНО! Только для своих веток.
Безопасный force
git push --force-with-lease
Не перезапишет если кто-то уже пушнул.
▶ GitHub CLI
Создать PR
gh pr create --title "Title" --body "Desc"
Squash merge
gh pr merge --squash <PR>

🔑 Авторизация

▶ SSH Keys
Генерация
ssh-keygen -t ed25519 -C "email@example.com"
Добавить в агент
eval "$(ssh-agent -s)" ssh-add ~/.ssh/id_ed25519
Проверить
ssh -T git@github.com
▶ Credentials
Кэш паролей (15 мин)
git config --global credential.helper cache
Хранить навсегда
git config --global credential.helper store
Использовать SSH вместо HTTPS
git config --global url."git@github.com:".insteadOf "https://github.com/"

📎 Submodules и Tags

▶ Submodules
Добавить
git submodule add URL <path>
Клонировать с submodule
git clone --recursive URL
Обновить
git submodule update --init --recursive
▶ Tags
Создать
git tag <v1.0.0>
Аннотированный
git tag -a <v1.0.0> -m "Release"
Удалить локальный
git tag -d <v1.0.0>
Удалить remote
git push origin --delete tag <v1.0.0>
Пушнуть и создать release
git tag <v1.0.0> && git push origin <v1.0.0>

✨ Лайфхаки

Быстрые alias
git config --global alias.st "status -s" git config --global alias.lg "log --oneline --graph"

Создаёт короткие команды: git st, git lg

Undo commit
git reset --soft HEAD~1

Отменяет коммит, файлы остаются в индексе

Проверить перед merge
git diff main..feature

Покажет все изменения до слияния

Squash через rebase
git rebase -i HEAD~3

Объединяет последние 3 коммита

Поиск потерянного
git reflog --all | grep "keyword"

Находит коммит даже после reset

Удалить неотслеживаемые
git clean -fd

Удаляет все новые файлы. ОПАСНО!

Пуш с upstream
git push -u origin feature

-u запоминает связь для будущих push

Remote URL
git remote -v

Показывает все remote репозитории

Изменить remote
git remote set-url origin URL

Меняет URL для origin

Проверить что игнорируется
git check-ignore -v <file>

Покажет почему файл игнорируется

Убрать assume-unchanged
git update-index --no-assume-unchanged <file>

Вернуть файл к отслеживанию

Squash merge локально
git merge --squash feature git commit

Объединяет все коммиты в один

Показать только коммиты в ветке
git log main..HEAD --oneline

Коммиты которые есть только здесь

dry-run перед опасным
git clean -fd --dry-run git reset --hard --dry-run

Покажет что будет удалено

Создать orphan branch
git checkout --orphan clean git commit -m "Initial"

Чистая история без коммитов
