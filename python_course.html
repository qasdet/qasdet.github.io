1. Основы Python

Use cases

  • Скрипты автоматизации: написать скрипт для переименования файлов, парсинга логов, резервного копирования.
  • CLI-утилиты: консольные программы с аргументами командной строки.
  • Прототипирование: быстрая проверка гипотез без настройки инфраструктуры.
  • Обработка данных: чтение CSV, фильтрация, запись результата.

Как это работает

Python — интерпретируемый язык. Исходный код компилируется в байт-код (файлы .pyc), который выполняется виртуальной машиной CPython. Динамическая типизация означает, что типы проверяются во время выполнения, а не при компиляции. Сборщик мусора (GC) автоматически освобождает память от объектов, на которые нет ссылок.

1.1 Переменные

Use cases

  • Хранение состояния: имя пользователя в сессии, счётчик запросов, флаги конфигурации.
  • Ссылки на объекты: переменная — это имя (метка), привязанная к объекту в памяти.
  • Множественное присваивание: a, b = b, a — обмен значениями без временной переменной.

Как это работает

В Python переменная — не ящик для значения, а имя (name), ссылающееся на объект. Операция x = 10 создаёт объект 10 (типа int) в куче и связывает с ним имя x. Если сделать x = 20, имя x перенаправляется на новый объект 20, а старый 10 будет удалён сборщиком мусора (GC), если на него больше нет ссылок.

a = 42
b = a      # b ссылается на тот же объект 42
a = 100    # a теперь ссылается на 100, b всё ещё на 42
print(b)   # 42

MAX_RETRIES = 5
PI = 3.14159

1.2 Базовые операторы

Use cases

  • Арифметика: расчёт скидок, конвертация валют, математические формулы.
  • Сравнение: фильтрация данных, проверка граничных условий.
  • Логические операторы: композиция условий, цепочки проверок.

Как это работает

Python поддерживает короткое замыкание (short-circuit) в логических операторах: and и or вычисляют правый операнд только если левый не определяет результат. Оператор // выполняет floor division — результат округляется вниз к отрицательной бесконечности: -7 // 3 == -3.

# Арифметика
a = 10 + 5    # 15
b = 10 - 5    # 5
c = 10 * 5    # 50
d = 10 / 3    # 3.333...
e = 10 // 3   # 3 (целочисленное)
f = 10 % 3    # 1 (остаток)
g = 2 ** 3    # 8 (степень)

# Сравнение
10 == 10  # True
10 != 5   # True
10 > 5    # True
10 <= 10  # True

# Логические
True and False  # False
True or False   # True
not True        # False

1.3 Ввод и вывод

Use cases

  • Интерактивные скрипты: запрос имени, пароля, пути к файлу.
  • Логирование: вывод отладочных сообщений, прогресс-баров, форматированных отчётов.
  • Конфигурация через stdin: передача параметров через пайпы в конвейере команд.

Как это работает

print() преобразует аргументы в строки (через __str__ или repr), разделяет их пробелом и выводит в sys.stdout с символом новой строки (параметр end). input() читает строку из sys.stdin до символа перевода строки и возвращает её без \n.

# Вывод
print("Hello, World!")
print(f"Name: {name}, Age: {age}")  # f-строки

# Ввод
user_input = input("Введите имя: ")
number = int(input("Введите число: "))

2. Типы данных и структуры

Use cases

  • Числа: финансовые расчёты (Decimal), научные вычисления (float), работа с датами (таймстемпы — int).
  • Строки: работа с текстом (поиск, замена, разметка, генерация отчётов).
  • Списки: очереди задач, буферы, упорядоченные коллекции элементов.
  • Словари: кеш (ключ → значение), конфигурация, счётчики частот.
  • Множества: удаление дубликатов, проверка принадлежности, пересечение аудиторий.
  • Enum: статусы заказов, роли пользователей, типы уведомлений.

Как это работает

Все типы данных в Python — объекты (даже int и bool). Неизменяемые типы (строки, кортежи, frozenset, числа) после создания нельзя модифицировать — любая «замена» создаёт новый объект. Изменяемые типы (списки, словари, множества) можно менять на месте.

Строки — это последовательности Unicode-символов, хранятся в памяти как массивы. Списки — динамические массивы указателей на PyObject. Словари — хэш-таблицы с усреднённым O(1) для вставки и поиска. Множества — также хэш-таблицы, но без значений, только ключи.

2.1 Числа

a: int = 42
b: float = 3.14
c: complex = 1 + 2j

from decimal import Decimal, getcontext
getcontext().prec = 10
price = Decimal("19.99")
total = price * 3

2.2 Строки

Use cases

  • Валидация: проверка email, номера телефона, пароля (длина, наличие цифр).
  • Парсинг: разбор CSV, JSON-строк, логов.
  • Шаблоны: f-строки для генерации HTML, SQL-запросов, писем.

Как это работает

Строка — неизменяемая последовательность Unicode-символов (в CPython — либо Latin-1, UCS-2, или UCS-4 в зависимости от старшего кода). Срезы s[i:j:k] создают новую строку за O(j-i). F-строки компилируются в код с вызовами __format__ для каждого выражения в фигурных скобках.

text = "Hello"
text.upper()
text.replace("l", "L")
text.split(",")

# f-строки (Python 3.6+)
name = "Alice"
msg = f"{name} is {age} years old"

2.3 Списки

Use cases

  • Очередь/стек: append()/pop() как LIFO; collections.deque для FIFO.
  • Сортировка и фильтрация: обработать коллекцию, отсортировать по ключу.
  • Матрицы: list of lists для таблиц, игровых полей, изображений.

Как это работает

Список — массив указателей на PyObject. При append() массив перераспределяется с запасом (over-allocation ~12.5%), чтобы амортизировать затраты. Вставка в начало insert(0, x) стоит O(n) — все элементы сдвигаются. List comprehension работает быстрее цикла for + append(), так как выполняет байт-код напрямую без вызова LIST_APPEND.

items = [1, 2, 3, 4, 5]
items.append(6)
items.pop()

# Срезы
nums = [0, 1, 2, 3, 4, 5]
nums[1:3]      # [1, 2]
nums[::-1]     # [5, 4, 3, 2, 1, 0]

# List comprehensions
squares = [x**2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]

2.4 Кортежи

Use cases

  • Неизменяемые ключи для словаря: координаты (x, y) как ключ.
  • Функции, возвращающие несколько значений: return min_val, max_val.
  • Данные, которые не должны меняться: конфигурационные константы, фиксированные наборы полей.

Как это работает

Кортеж — неизменяемая последовательность, реализованная как массив указателей. Неизменяемость означает, что ссылки внутри кортежа нельзя перезаписать (но если элемент кортежа — изменяемый объект, его содержимое менять можно). namedtuple генерирует класс с легковесными экземплярами (без __dict__), экономя память.

point = (10, 20)
x, y = point

from collections import namedtuple
Person = namedtuple("Person", ["name", "age"])
p = Person("Alice", 30)
p.name

2.5 Словари

Use cases

  • Кеширование: запоминать результат дорогой операции по ключу.
  • Конфигурация: читать config.json как вложенный словарь.
  • Счётчики: подсчёт частоты символов, слов, элементов.
  • ORM / маппинг: поля БД → атрибуты модели.

Как это работает

Словарь — хэш-таблица. Каждый ключ хэшируется (hash()), хэш маппируется в индекс массива. При коллизиях используется open addressing (открытая адресация). В Python 3.6+ словари сохраняют порядок вставки (стали компактными — две таблицы: одна для индексов, вторая для данных). defaultdict перехватывает KeyError в __missing__ и вставляет значение по умолчанию.

user = {"name": "Alice", "age": 30}
user.get("phone", "N/A")
user.update({"age": 31})

# Dict comprehensions
squares = {x: x**2 for x in range(5)}

# Defaultdict
from collections import defaultdict
counts = defaultdict(int)
counts["a"] += 1

2.6 Множества

Use cases

  • Уникальные элементы: получить неповторяющиеся значения из списка.
  • Операции над группами: поиск общих друзей, разность аудиторий.
  • Проверка принадлежности: быстрый in — O(1).

Как это работает

Множество — хэш-таблица (как словарь, но без значений). Поиск in, вставка add(), удаление remove() — в среднем O(1). Элементы должны быть хэшируемы (неизменяемые). frozenset — неизменяемая версия, может быть элементом другого множества.

a = {1, 2, 3, 4, 5}
b = {4, 5, 6, 7, 8}
a | b   # объединение
a & b   # пересечение
a - b   # разность

2.7 None

Use cases

  • Отсутствие значения: возврат из функции, если ничего не найдено.
  • Опциональные параметры: аргумент по умолчанию None, чтобы отличить «не передан» от «передан falsy».
  • Сигнализация ошибки в типе-возврате: Optional[T].

Как это работает

None — синглтон (единственный экземпляр NoneType). Сравнение всегда через is None, а не == None, так как is проверяет идентичность объектов (по адресу в памяти), а == может быть перегружен и работать медленнее.

value = None
if value is None:
    print("No value")

2.10 Копирование: поверхностное vs глубокое

import copy

original = {"items": [1, 2, 3], "name": "test"}

# Поверхностное: вложенные объекты не копируются
shallow = copy.copy(original)
shallow["items"].append(4)
print(original["items"])  # [1, 2, 3, 4] — изменился!

# Глубокое: полная независимая копия
deep = copy.deepcopy(original)
deep["items"].append(5)
print(original["items"])  # [1, 2, 3, 4] — не изменился

# Срезы списков — тоже поверхностная копия
a = [[1, 2], [3, 4]]
b = a[:]           # поверхностная
b[0].append(99)
print(a[0])         # [1, 2, 99]

3. Управляющие конструкции

Use cases

  • Условия: проверка прав доступа, валидация данных, маршрутизация запросов.
  • Match-case (3.10+): парсинг AST, обработка команд, разбор структурированных данных.
  • Циклы: итерация по строкам файла, пакетная обработка записей, pagination через API.

Как это работает

Интерпретатор выполняет управляющие конструкции последовательно. if/elif/else компилируется в условные переходы в байт-коде (POP_JUMP_IF_FALSE). match-case добавлен в Python 3.10 (PEP 634–636) — компилируется в цепочку сравнений с распаковкой паттернов. Цикл for использует протокол итератора: вызывает __iter__() один раз и __next__() на каждой итерации до StopIteration.

3.1 Условный оператор if

age = 18
if age < 18:
    print("Minor")
elif age == 18:
    print("Just adult")
else:
    print("Adult")

status = "adult" if age >= 18 else "minor"

3.2 Match-case (Python 3.10+)

Use cases

  • Командный парсер: разбор действий [action, *args].
  • JSON-маршрутизация: обработка разных типов событий (webhook).
  • State machine: управление состояниями по паттерну (current_state, event).

Как это работает

match сравнивает subject с шаблонами. Паттерны могут быть: литералы (1, "a", None), значения из enum, распаковка последовательностей, привязка к переменным, классовые иерархии, стражники case _ if condition. Mypy поддерживает match-case с анализом типов. Компилятор преобразует match-case в эффективный байт-код без длинных if-elif.

def process(value: str | int | None) -> str:
    match value:
        case str():
            return value.upper()
        case int():
            return str(value)
        case None:
            return "empty"
        case _:
            return "unknown"

3.3 Циклы

Use cases

  • Пакетная обработка: прочитать 1000 записей из API по 100 за запрос (while).
  • Обработка файлов: for line in file: — лениво, строка за строкой.
  • Enumerate: получить индекс и элемент одновременно (замена range(len(...))).

Как это работает

for в Python — for-each, не C-стиль. Он получает итератор и на каждом шаге вызывает next(). range() не создаёт список — это объект, генерирующий числа лениво. while True + break — идиома «бесконечный цикл до условия». else у цикла выполняется только если цикл завершён без break.

for i in range(5):
    print(i)

for i, item in enumerate(["a", "b", "c"]):
    print(i, item)

count = 0
while count < 5:
    print(count)
    count += 1

for n in range(10):
    if n == 7:
        break
else:
    print("Loop completed")  # не выполнится

4. Функции

Use cases

  • Разделение логики: каждая функция — один шаг в пайплайне.
  • Переиспользование: написать один раз, вызывать из разных модулей.
  • Абстракция: скрыть сложность за понятным именем.
  • Композиция: собрать сложное поведение из маленьких функций.

Как это работает

Функции — объекты первого класса (first-class citizens). Их можно присваивать, передавать, возвращать. При определении def f(): ... создаётся объект функции (с атрибутами __code__, __defaults__, __closure__). При вызове создаётся кадр (frame) в стеке вызовов с локальными переменными. Аннотации типов (type hints) не влияют на выполнение — они хранятся в __annotations__ и проверяются внешними инструментами (mypy, Pyright).

4.1 Определение функций

Use cases

  • Типизированные функции: самодокументируемый код с явным контрактом.
  • Значения по умолчанию: опциональные параметры без перегрузки.
  • args/*kwargs: обработчики событий, декораторы, обёртки.

Как это работает

Параметры со значениями по умолчанию вычисляются один раз при определении функции, а не при каждом вызове. Поэтому mutable default (например, def f(x=[])) ведёт к неожиданному поведению — список общий для всех вызовов. / и * в синтаксисе параметров — PEP 570 и PEP 3102 — разделяют позиционные, позиционно-именованные и строго именованные аргументы.

def greet(name: str) -> str:
    return f"Hello, {name}!"

def power(base: float, exp: int = 2) -> float:
    return base ** exp

def func(a, b, /, c, d, *, e, f):
    pass

def log(message: str, *args, **kwargs):
    print(f"{message}: args={args}, kwargs={kwargs}")

4.2 Области видимости

Use cases

  • Замыкания: фабрики функций, декораторы.
  • Глобальные константы: но лучше избегать — использовать конфигурацию.
  • nonlocal: счётчики, накопление состояния внутри вложенной функции.

Как это работает

LEGB — порядок поиска имени: Local (внутри функции), Enclosing (внешняя функция), Global (модуль), Built-in (встроенные имена). nonlocal поднимает имя на один уровень enclosing (но не до global). global — в пространство имён модуля. Без nonlocal присваивание outer_var = 1 внутри inner() создаст новую локальную переменную, не тронув внешнюю.

global_var = 10

def outer():
    outer_var = 20

    def inner():
        nonlocal outer_var
        global global_var
        outer_var += 1
        global_var += 1

    return inner()

4.3 Анонимные функции (lambda)

Use cases

  • Сортировка по ключу: sorted(items, key=lambda x: x.field).
  • Коллбеки в GUI/Tkinter: button.config(command=lambda: handler()).
  • Адаптация интерфейса: преобразование типов встроенными функциями.

Как это работает

lambda — синтаксический сахар для простой функции. lambda x: x * 2 эквивалентно def f(x): return x * 2. Отличие: lambda — выражение (можно в inline), может содержать только одно выражение. Возвращает объект-функцию.

square = lambda x: x ** 2
evens = list(filter(lambda x: x % 2 == 0, nums))
sorted_by_len = sorted(["aaa", "bb", "c"], key=lambda s: len(s))

4.4 Замыкания

Use cases

  • Фабрики функций: создать несколько вариаций одной логики.
  • Счётчики и накопители: инкапсулировать состояние без класса.
  • Декораторы: замыкание — основа декоратора.

Как это работает

Замыкание — функция, захватившая переменные из enclosing scope. Технически: функция хранит ссылку на объект __closure__ — кортеж ячеек (cell objects), каждая ссылается на захваченную переменную. Даже когда внешняя функция завершилась, ячейки живут, пока существует внутренняя функция.

def make_multiplier(factor: int):
    def multiply(number: int) -> int:
        return number * factor
    return multiply

double = make_multiplier(2)
print(double(5))   # 10

4.5 Type hints для функций

Use cases

  • IDE-автодополнение: PyCharm, VS Code подсказывают типы.
  • Статический анализ: mypy/Pyright находят ошибки типов до запуска.
  • Документация: типы — часть контракта функции.

Как это работает

def range_gen(n: int) -> Generator[int, None, None]:

    while i < n:
        yield i
        i += 1

        value = yield total       # получает значение через send()

            break
        total += value
    return f"Final: {total}"

except StopIteration as e:
    print(e.value)                # "Final: 60"

            yield from flatten(item)
        else:
            yield item

4.14 Декораторы: продвинутые техники

from functools import wraps
from typing import Any, Callable, ParamSpec, TypeVar
import time
import logging

P = ParamSpec("P")
T = TypeVar("T")

# Декоратор с аргументами и сохранением типов
def retry(
    max_attempts: int = 3,
    delay: float = 0.1,
    exceptions: tuple[type[Exception], ...] = (Exception,),
) -> Callable[[Callable[P, T]], Callable[P, T]]:
    """Повторяет вызов при исключении."""
    def decorator(func: Callable[P, T]) -> Callable[P, T]:
        @wraps(func)
        def wrapper(*args: P.args, **kwargs: P.kwargs) -> T:
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except exceptions:
                    if attempt == max_attempts - 1:
                        raise
                    time.sleep(delay)
            raise RuntimeError("Unreachable")
        return wrapper
    return decorator

# Декоратор-класс (через __call__)
class Timed:
    """Декоратор как класс."""
    def __init__(self, func: Callable) -> None:
        self.func = func

    def __call__(self, *args, **kwargs) -> Any:
        start = time.time()
        result = self.func(*args, **kwargs)
        print(f"{self.func.__name__}: {time.time() - start:.4f}s")
        return result

@Timed
def slow() -> None:
    time.sleep(0.5)

# Декоратор для методов с доступом к self
def log_calls(func: Callable) -> Callable:
    @wraps(func)
    def wrapper(self, *args, **kwargs):
        logging.info(f"{self.__class__.__name__}.{func.__name__} called")
        return func(self, *args, **kwargs)
    return wrapper

# Декоратор с опциональными скобками
def optional_parenthesis(func=None, *, enabled: bool = True):
    """@decorator и @decorator() работают одинаково."""
    def decorator(f):
        @wraps(f)
        def wrapper(*args, **kwargs):
            if enabled:
                print(f"Calling {f.__name__}")
            return f(*args, **kwargs)
        return wrapper
    if func is None:
        return decorator
    return decorator(func)

@optional_parenthesis
def f1(): pass

@optional_parenthesis()
def f2(): pass

# Каррирование (частичное применение)
from functools import partial, reduce

def curry(func: Callable) -> Callable:
    """Преобразует функцию в каррированную форму."""
    def _curry(*args, **kwargs):
        if len(args) + len(kwargs) >= func.__code__.co_argcount:
            return func(*args, **kwargs)
        return lambda *a, **kw: _curry(*args, *a, **kwargs, **kw)
    return _curry

@curry
def add3(a: int, b: int, c: int) -> int:
    return a + b + c

print(add3(1)(2)(3))       # 6
print(add3(1, 2)(3))       # 6

# Мемоизация (ручная)
def memoize(func: Callable[P, T]) -> Callable[P, T]:
    """Кеширует результаты вызовов."""
    cache: dict[tuple, Any] = {}

    @wraps(func)
    def wrapper(*args: P.args, **kwargs: P.kwargs) -> T:
        key = (args, tuple(sorted(kwargs.items())))
        if key not in cache:
            cache[key] = func(*args, **kwargs)
        return cache[key]

    wrapper.cache = cache
    wrapper.cache_clear = cache.clear
    return wrapper

@memoize
def expensive(n: int) -> int:
    print(f"Computing {n}...")
    return n * n

print(expensive(5))   # Computing 5... 25
print(expensive(5))   # 25 (из кеша)

6. Классы и ООП

Use cases

  • Модели данных: User, Order, Product — сущности бизнес-логики.
  • Абстракция API: скрыть детали реализации за интерфейсом класса.
  • Полиморфизм: единый интерфейс для разных реализаций (платежи, нотификации).
  • Фреймворки: Django ORM, FastAPI, SQLAlchemy — всё построено на классах.

Как это работает

Класс — шаблон для создания объектов. При определении класса Python создаёт объект-тип (также класса type), с методами, атрибутами, MRO. При вызове MyClass() вызывается __new__ (создаёт экземпляр) и __init__ (инициализирует). Методы экземпляра получают self автоматически — Python передаёт экземпляр первым аргументом. @classmethod получает cls, @staticmethod не получает ни self ни cls.

6.1 Базовый класс

Use cases

  • DTO: объекты для передачи данных между слоями.
  • Сервисы: бизнес-логика в методах.
  • Фабрики: classmethod как альтернативный конструктор.

Как это работает

__init__ — не конструктор, а инициализатор. Конструктор — __new__ (выделяет память). __repr__ должен возвращать однозначное воспроизводимое представление (часто как код для создания объекта). __str__ — человекочитаемое представление, print() использует __str__, а fallback — __repr__.

class User:
    total_users = 0

    def __init__(self, name: str, email: str):
        self.name = name
        self.email = email
        User.total_users += 1

    def __repr__(self) -> str:
        return f"User(name={self.name!r}, email={self.email!r})"

    def __str__(self) -> str:
        return f"{self.name} <{self.email}>"

    @classmethod
    def create_anonymous(cls) -> "User":
        return cls("Anonymous", "anon@example.com")

    @staticmethod
    def validate_email(email: str) -> bool:
        return "@" in email

6.2 Наследование

Use cases

  • Иерархия сущностей: User → Admin, Employee, Customer.
  • Расширение фреймворка: наследовать Model из Django ORM.
  • Переиспользование: общая логика в родителе, специфика — в наследниках.

Как это работает

super() возвращает прокси-объект, следующий MRO (Method Resolution Order — порядок разрешения методов, вычисленный по C3-линеаризации). MRO гарантирует, что каждый класс вызывается до своих родителей и что порядок предшествования соблюдён. D.__mro__ показывает последовательность поиска методов.

class Admin(User):
    def __init__(self, name: str, email: str, role: str = "admin"):
        super().__init__(name, email)
        self.role = role

# MRO: D → B → C → A → object
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass
print(D.__mro__)

6.3 Инкапсуляция

Use cases

  • Скрытие состояния: _balance — только через методы deposit/withdraw.
  • Защищённые данные: пароли, токены, внутренние идентификаторы.
  • Контролируемый доступ: геттеры/сеттеры с валидацией.

Как это работает

Python не имеет strict private, как Java. _ — соглашение (protected, не трогать снаружи). __ (dunder) включает name mangling: __pin преобразуется в _BankAccount__pin. Это предотвращает случайное переопределение в подклассах. Настоящей приватности нет — имя всё ещё доступно через _ClassName__attr.

class BankAccount:
    def __init__(self, owner: str, balance: float = 0):
        self.owner = owner
        self._balance = balance
        self.__pin = "1234"

    @property
    def balance(self) -> float:
        return self._balance

    def deposit(self, amount: float):
        if amount <= 0:
            raise ValueError("Amount must be positive")
        self._balance += amount

6.4 Магические методы

Use cases

  • Арифметика: Vector + Vector, Matrix * Scalar.
  • Сравнение: User == User по id, Date < Date.
  • Итерация: объект можно использовать в for x in obj.
  • Контекстные менеджеры: __enter__ / __exit__.

Как это работает

Магические методы — часть протокола Python. __add__ вызывается при +, __eq__ при ==. Если __eq__ не определён, сравниваются id объектов. return NotImplemented (не путать с NotImplementedError) позволяет Python попробовать зеркальный метод у другого операнда.

class Vector:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    def __add__(self, other: "Vector") -> "Vector":
        return Vector(self.x + other.x, self.y + other.y)

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Vector):
            return NotImplemented
        return self.x == other.x and self.y == other.y

v1 = Vector(1, 2)
v2 = Vector(3, 4)
print(v1 + v2)     # Vector(4, 6)

Use cases

  • Платёжные шлюзы: PaymentGateway.charge() — разные реализации.
  • Репозитории: контракт на CRUD-операции.
  • Уведомления: Notification.send() — email, SMS, push.

Как это работает

ABC (Abstract Base Class) запрещает создание экземпляра, если не реализованы все абстрактные методы. При создании объекта Python проверяет __abstractmethods__. Если список не пуст — TypeError. Отличие от Protocol: ABC требует явного наследования, Protocol — структурное соответствие.

7. Dataclasses

Use cases

  • DTO (Data Transfer Objects): передача данных между слоями приложения.
  • Value Objects: неизменяемые объекты-значения (деньги, координаты, адрес).
  • Конфигурация: структура с параметрами с умолчаниями.
  • Модели данных: замена ручного init с большим числом полей.

Как это работает

@dataclass — декоратор (PEP 557, Python 3.7+). Он читает аннотации класса и автоматически генерирует __init__, __repr__, __eq__, __hash__, __lt__ (если order=True). Имена полей берутся из аннотаций типов. Поля со значениями — необязательные. field() даёт тонкую настройку: дефолты, исключение из __repr__ или __eq__, init=False (вычисляемое поле).

7.1 Основы

Use cases

  • API-модели: сериализация запросов/ответов.
  • Сложные ключи: словарные ключи, требующие __hash__ и __eq__.
  • Отладка: __repr__ автоматически показывает все поля.

Как это работает

Генерация методов происходит на этапе создания класса — в __init_subclass__. Порядок полей совпадает с порядком в классе. Поля без значения по умолчанию идут до полей со значениями (иначе TypeError). field(default_factory=list) — каждый экземпляр получает новый пустой список, избегая проблемы mutable default.

from dataclasses import dataclass, field, asdict

@dataclass
class User:
    name: str
    email: str
    age: int = 0
    tags: list[str] = field(default_factory=list)

user = User("Alice", "alice@example.com", 30)
print(asdict(user))

7.2 Замороженные dataclasses (неизменяемые)

Use cases

  • Value Objects: Currency, Money, DateTimeRange.
  • Иммутабельные конфиги: нельзя случайно изменить после создания.
  • Хэшируемые объекты: можно использовать как ключ словаря.

Как это работает

frozen=True — после __init__ перехватывается попытка присваивания атрибута через __setattr__, вызывая FrozenInstanceError. Это поверхностная заморозка: если поле содержит список, его содержимое менять можно. Для глубокой нужны неизменяемые типы (tuple вместо list).

@dataclass(frozen=True)
class Point:
    x: float
    y: float

    def distance_to(self, other: "Point") -> float:
        return ((self.x - other.x) ** 2 + (self.y - other.y) ** 2) ** 0.5

8. Protocols (структурная типизация)

Use cases

  • Duck typing с проверкой: «Если объект имеет метод save(), он — Repository».
  • Слабая связанность: класс не обязан знать о родителях, просто реализовать интерфейс.
  • Тестирование: легко подменить реальный класс моком с тем же протоколом.
  • Старый код: добавить типизацию без изменения иерархии наследования.

Как это работает

Protocol (PEP 544, Python 3.8+) — это структурная подтипизация. Класс удовлетворяет протоколу, если у него есть все методы/атрибуты с нужными сигнатурами — явное наследование не требуется. Mypy/Pyright проверяет это статически. @runtime_checkable добавляет поддержку isinstance() — проверку наличия методов во время выполнения.

8.1 Основы

Use cases

  • Репозиторий: любой класс с find_by_id и save — Repository.
  • Стратегия: любой класс с execute(data) — Strategy.
  • Адаптер: любой класс с read()/write() — IO-устройство.

Как это работает

В отличие от ABC (номинальная типизация — «кто наследует»), Protocol проверяет структуру: «у кого есть метод find_by_id(self, id: str) -> dict | None». Python не проверяет это в рантайме (кроме isinstance с @runtime_checkable). Mypy анализирует все вызовы и проверяет, что аргумент имеет нужные методы.

from typing import Protocol

class Repository(Protocol):
    def find_by_id(self, id: str) -> dict | None: ...
    def save(self, entity: dict) -> dict: ...

class UserRepository:
    def find_by_id(self, id: str) -> dict | None:
        return {"id": id, "name": "Alice"}

    def save(self, entity: dict) -> dict:
        return entity

def process_entity(repo: Repository, id: str) -> None:
    entity = repo.find_by_id(id)
    if entity:
        print(f"Found: {entity}")

repo = UserRepository()
process_entity(repo, "1")

9. Декораторы

Use cases

  • Логирование: замер времени, аргументы, результат вызова.
  • Кеширование: запомнить результат дорогой функции (lru_cache).
  • Контроль доступа: проверка прав, аутентификация.
  • Retry: повтор при временных ошибках.
  • Валидация: проверка аргументов перед вызовом.

Как это работает

Декоратор — функция, принимающая другую функцию и возвращающая новую. Синтаксис @decorator — сахар: @d f равносильно f = d(f). Стек декораторов: @a @b f → a(b(f)). @wraps(func) (из functools) копирует метаданные (__name__, __doc__) из исходной функции в обёртку.

9.1 Декораторы функций

Use cases

  • Профилирование: измерить, сколько времени выполняется функция.
  • Трассировка: логировать вызовы с аргументами.
  • Отложенная инициализация: ресурс создаётся при первом вызове.

Как это работает

@timing заменяет slow_function на wrapper. wrapper имеет доступ к оригинальной slow_function через замыкание (имя func). @wraps(func) копирует __wrapped__, __name__, __qualname__, __doc__, __dict__, __module__ из func в wrapper.

from functools import wraps
import time

def timing(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end - start:.4f}s")
        return result
    return wrapper

@timing
def slow_function():
    time.sleep(0.5)
    return "done"

9.2 Декораторы с аргументами

Use cases

  • Retry с настройкой: количество попыток, задержка, какие ошибки ловить.
  • Rate limit: ограничение вызовов в секунду.
  • Кеш с TTL: время жизни кеша.

Как это работает

Декоратор с аргументами — функция, возвращающая декоратор. @retry(3) → сначала retry(3) вызывается, возвращает декоратор, потом этот декоратор применяется к функции. Три уровня: функция-фабрика → декоратор → wrapper.

def retry(max_attempts: int = 3, delay: float = 0.1):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_attempts - 1:
                        raise
                    time.sleep(delay)
            return None
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.2)
def unstable_network_call():
    pass

9.4 Декораторы классов

Use cases

  • Singleton: один экземпляр класса на всё приложение.
  • Регистрация: автоматически зарегистрировать класс в реестре.
  • Добавление атрибутов: например, @dataclass.

Как это работает

Декоратор класса получает класс и возвращает новый класс (или модифицирует переданный). @singleton заменяет класс на функцию-фабрику, которая каждый раз проверяет instances и возвращает существующий экземпляр.

def singleton(cls):
    instances = {}

    @wraps(cls)
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    return get_instance

@singleton
class Config:
    def __init__(self):
        self.value = 42

c1 = Config()
c2 = Config()
print(c1 is c2)  # True

10. Контекстные менеджеры

Use cases

  • Управление ресурсами: файлы, сокеты, соединения с БД — гарантированно закрыть.
  • Транзакции: commit при успехе, rollback при ошибке.
  • Блокировки: захватить и освободить мьютекс.
  • Изменение состояния временно: переключение директории, env-переменные.

Как это работает

Протокол контекстного менеджера: __enter__ → выполнить код до yield → при выходе (нормальном или по исключению) → __exit__. Если __exit__ возвращает True, исключение подавляется. Исключение передаётся через аргументы __exit__(exc_type, exc_val, exc_tb). @contextmanager превращает генератор с yield в контекстный менеджер.

10.1 Синтаксис with

with open("file.txt", "r") as f:
    content = f.read()

10.2 Классовый контекстный менеджер

Use cases

  • Сетевые соединения: connect() в __enter__, close() в __exit__.
  • Транзакции БД: __exit__ проверяет исключение и делает commit/rollback.
  • Профилирование: засечь время в __enter__, вывести в __exit__.

Как это работает

__enter__ возвращает объект, связанный с as. Если as не используется, возврат игнорируется. __exit__ вызывается всегда, даже при исключении. return False → исключение пробрасывается дальше. return True → исключение съедено.

class FileProcessor:
    def __init__(self, filename: str, mode: str = "r"):
        self.filename = filename
        self.mode = mode
        self.file = None

    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file

    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.file:
            self.file.close()
        return False

10.3 Контекстный менеджер через contextmanager

Use cases

  • Быстрая обёртка: @contextmanager — без создания класса.
  • Транзакции, блокировки, паузы: короткая логика.

Как это работает

@contextmanager из contextlib — декоратор для генератора. Код до yield — __enter__, сам yield — точка возврата, код после yield — __exit__. Если в блоке with исключение — оно выбрасывается в генератор в точке yield. Генератор может его поймать (начиная с Python 2.5 через .throw()).

from contextlib import contextmanager

@contextmanager
def database_transaction(db):
    try:
        print("BEGIN TRANSACTION")
        yield
        db.commit()
    except Exception:
        db.rollback()
        raise

with database_transaction(db):
    db.execute("INSERT INTO users ...")

11. Генераторы и итераторы

Use cases

  • Ленивая загрузка: чтение большого файла строками, не загружая в память.
  • Потоковая обработка: бесконечный поток данных с фильтрацией.
  • Пагинация через API: итерация по страницам, загрузка «на лету».
  • Генерация последовательностей: ID, таймстемпы, пароли.

Как это работает

Итератор — объект с __iter__() (возвращает себя) и __next__() (возвращает следующий элемент или StopIteration). Генератор — функция с yield. При вызове возвращается объект-генератор. При первом next() выполняется код до первого yield, значение передаётся вызывающему, состояние замораживается. При следующем next() — продолжение с места остановки. Генераторы — основа асинхронности в Python (до появления async/await).

11.1 Генераторы (yield)

Use cases

  • Последовательности Фибоначчи, простые числа: бесконечный ряд.
  • Ленивый фильтр: обход дерева файловой системы.
  • Пайплайны: цепочки генераторов для обработки данных.

Как это работает

Локальные переменные и счётчик команд сохраняются в frame-объекте генератора. Генератор приостанавливает выполнение на yield и возобновляет с того же места. Генератор — однократный: после исчерпания (StopIteration) новые элементы не выдаёт. return в генераторе — значение помещается в StopIteration.value (доступно через StopIteration.value или yield from).

def count_up_to(n: int):
    i = 1
    while i <= n:
        yield i
        i += 1

for num in count_up_to(5):
    print(num)

def read_large_file(filename: str):
    with open(filename, "r") as f:
        for line in f:
            yield line.strip()

for line in read_large_file("huge_file.txt"):

11.3 Генераторные выражения

Use cases

  • Сумма, any, all: sum(x**2 for x in range(10)) — без создания списка.
  • Первый подходящий: next(x for x in items if condition).
  • Конвейеры: цепочка фильтров без промежуточных списков.

Как это работает

(x**2 for x in range(10)) — анонимный генератор. Разница с list comprehension: [] строит весь список в памяти, () создаёт генератор. sum(), any(), all() принимают итератор — можно передать generator expression без скобок: sum(x for x in range(10)).

11.4 yield from

Use cases

  • Композиция генераторов: объединить несколько источников данных.
  • Рекурсивный обход: yield from для плоской выдачи вложенных структур.
  • Упрощение делегирования: вместо цикла for над вложенным генератором.

Как это работает

yield from generator — синтаксический сахар для for item in generator: yield item. yield from также устанавливает двустороннюю связь: send(), throw(), close() пробрасываются в делегированный генератор.

def merged_generator():

    yield from generator2()

12. Асинхронное программирование

Use cases

  • HTTP-сервер: FastAPI, aiohttp — тысячи одновременных соединений.
  • Микросервисы: асинхронные RPC, очередь задач, фоновые воркеры.
  • WebSockets: чаты, стриминг данных, real-time уведомления.
  • I/O-bound операции: запросы к API, чтение файлов, БД (asyncpg, aiomysql).

Как это работает

asyncio использует event loop (цикл событий) на базе selectors (epoll/kqueue/IOCP). Корутина — генератор, обёрнутый в async def. await — точка, где event loop может переключиться на другую задачу. GIL не блокируется, но CPU-bound операции блокируют event loop — нужны отдельные процессы или потоки. TaskGroup (3.11+) управляет группой задач с гарантированной обработкой исключений.

12.1 Основы asyncio

Use cases

  • Параллельные HTTP-запросы: fetch несколько URL одновременно.
  • Ожидание БД: asyncpg, SQLAlchemy async — не блокировать сервер.
  • Фоновые задачи: периодическая очистка кеша, отправка писем.

Как это работает

async def — корутина. Вызов fetch_user("1") возвращает объект корутины, но не выполняет код. Выполнение начинается, когда корутина передана в event loop. await приостанавливает корутину до завершения асинхронной операции. asyncio.run(main()) создаёт event loop, выполняет main(), и закрывает loop. asyncio.gather(*tasks) запускает задачи конкурентно и ждёт все.

import asyncio

async def fetch_user(user_id: str) -> dict:
    await asyncio.sleep(0.1)
    return {"id": user_id, "name": f"User{user_id}"}

async def main():
    tasks = [fetch_user(uid) for uid in ["1", "2", "3"]]
    results = await asyncio.gather(*tasks)
    print(results)

asyncio.run(main())

Как это работает

class AsyncCounter:

        self.current = 0

12.5 TaskGroup (Python 3.11+)

Use cases

  • Структурированная конкурентность: все задачи созданы и завершены в одном контексте.
  • Обработка ошибок: если одна задача упала — отменить остальные.

Как это работает

TaskGroup — структурированная альтернатива gather. При входе создаётся контекст. create_task() запускает корутину. При выходе group ждёт все задачи. Если хотя бы одна упала — все остальные отменяются (cancel()) и исключение выбрасывается из __aexit__.

async def main():

        task1 = tg.create_task(fetch_user("1"))
        task2 = tg.create_task(fetch_user("2"))

    print(task1.result())

13.3 Any, TypeVar, Generic

Use cases

  • Any: когда тип правда неизвестен (данные из JSON, input).
  • TypeVar: чтобы first() сохраняла тип: first([1, 2]) → int.
  • Generic: абстрактный контейнер (Stack, Queue) для любого типа.

Как это работает

T = TypeVar("T") — переменная типа. def first(items: list[T]) -> T — при вызове first([1, "a"]) mypy выведет T = int | str. Generic[T] — базовый класс для дженериков. bound= ограничивает: T must be SupportsLess или его подтип.

from typing import TypeVar, Generic

T = TypeVar("T")

def first(items: list[T]) -> T | None:
    return items[0] if items else None

class Stack(Generic[T]):
    def __init__(self):
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

stack: Stack[int] = Stack()
stack.push(1)
val = stack.pop()

14. Обработка ошибок и исключения

Use cases

  • Валидация: клиент прислал неверные данные — ValidationError.
  • Сетевые ошибки: таймаут, соединение разорвано — ConnectionError.
  • Бизнес-логика: недостаточно средств, пользователь не найден — DomainError.
  • Обработка отказов: попробовать ещё раз, переключиться на резервный сервер.

Как это работает

Исключения — объекты, наследующие BaseException (реже) или Exception (почти всегда). При raise интерпретатор разматывает стек (stack unwinding): ищет except, проходя через кадры вызовов. Если не находит — программа падает с traceback. raise ... from — цепочка исключений (__cause__ / __context__). ExceptionGroup (3.11+) — группа исключений, обрабатывается через except*.

14.1 Базовый try/except

Use cases

  • Try-except-finally: ресурс должен быть закрыт всегда (finally).
  • Try-except-else: «если не было ошибки, сделать X».
  • Несколько except: разные реакции на разные типы ошибок.

Как это работает

else — выполняется если не было ни одного except. finally — выполняется всегда (даже если return, break, continue). Если и except и finally содержат return — побеждает finally. as e связывает экземпляр исключения с переменной.

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Деление на ноль")
except (ValueError, TypeError) as e:
    print(f"Ошибка: {e}")
except Exception:
    print("Неизвестная ошибка")
else:
    print("Ошибок не было")
finally:
    print("Выполняется всегда")

14.2 Пользовательские исключения

Use cases

  • DomainError: бизнес-правила (недостаточно средств, заказ не может быть отменён).
  • ApiError: ошибки HTTP (404, 403, 500) с кодом и сообщением.
  • ValidationError: ошибка с указанием поля (field: "email", message: "invalid").

Как это работает

Пользовательские исключения — подклассы Exception. super().__init__(msg) передаёт сообщение. Атрибуты (.field, .user_id) позволяют передать контекст. Обработчик может поймать DomainError (все доменные ошибки) или конкретный UserNotFoundError.

class ValidationError(DomainError):
    def __init__(self, field: str, message: str):
        self.field = field
        self.message = message
        super().__init__(f"{field}: {message}")

14.5 Exception Groups (Python 3.11+)

Use cases

  • Пакетная валидация: собрать все ошибки валидации формы и показать сразу.
  • Множественные сбои: при вызове TaskGroup — одна задача упала, другая тоже.

Как это работает

ExceptionGroup — исключение, содержащее список исключений. except* ValueError — обработчик для всех ValueError внутри группы. Если после всех except* остались необработанные — выбрасывается новая группа. Нельзя смешивать except и except* в одном try.

        errors.append(ValueError("Name is required"))
    if "email" not in data:
        errors.append(ValueError("Email is required"))
    if errors:
        raise ExceptionGroup("Validation failed", errors)

try:
    validate_user({"name": "Alice"})
except* ValueError as eg:
    for exc in eg.exceptions:
        print(f"ValueError: {exc}")

15. Свойства (Property)

Use cases

  • Валидация при присваивании: email, возраст, пароль — проверить при user.email = ....
  • Вычисляемые поля: full_name, age (из даты рождения), area.
  • Ленивая инициализация: ресурс создаётся при первом обращении.
  • Read-only API: user.id — можно прочитать, нельзя изменить.

Как это работает

@property — декоратор, превращающий метод в атрибут только для чтения. Создаётся объект property(fget, fset, fdel, doc). @x.setter и @x.deleter — декораторы, вызывающие property.setter/deleter для переопределения методов. cached_property (functools, Python 3.8+) — вычисляется один раз, результат сохраняется в __dict__ экземпляра.

15.1 @property

Use cases

  • Объекты с валидацией: product.price = -5 — ошибка.
  • Вычисляемые атрибуты: employee.salary хранит базовую ставку, employee.total_compensation — вычисляет с бонусом.
  • Backward compatibility: заменить публичное поле на property без изменения интерфейса.

Как это работает

@property создаёт дескриптор (descriptor protocol). Обращение obj.email → type(obj).__dict__['email'].__get__(obj, type(obj)). Присваивание obj.email = val → type(obj).__dict__['email'].__set__(obj, val). Удаление del obj.email → __delete__.

class User:
    def __init__(self, name: str):
        self._name = name
        self._email = None

    @property
    def name(self) -> str:
        return self._name

    @property
    def email(self) -> str | None:
        return self._email

    @email.setter
    def email(self, value: str) -> None:
        if "@" not in value:
            raise ValueError("Invalid email")
        self._email = value

    @email.deleter
    def email(self) -> None:
        self._email = None

user = User("Alice")
user.email = "alice@example.com"

15.2 cached_property

Use cases

  • Тяжёлые вычисления: загрузка конфига, парсинг большого файла, агрегация данных.
  • Однократные операции: запрос к БД при первом обращении, потом из кеша.

Как это работает

cached_property — дескриптор, который после первого вычисления заменяет себя в instance.__dict__ на результат. При повторном обращении Python находит значение в __dict__ (быстрее, чем дескриптор). Не работает с __slots__ и не поддерживает setter.

from functools import cached_property

class DataProcessor:
    def __init__(self, data: list[int]):
        self.data = data

    @cached_property
    def computed(self) -> dict:
        print("Computing...")
        return {
            "sum": sum(self.data),
            "avg": sum(self.data) / len(self.data),
        }

dp = DataProcessor([1, 2, 3, 4, 5])
print(dp.computed)
print(dp.computed)  # из кеша

Use cases

  • Конвейеры обработки: цепочка map → filter → reduce над коллекцией.
  • Неизменяемость потоков данных: безопасно передавать между процессами/потоками.
  • Композиция логики: собрать сложную обработку из маленьких чистых функций.
  • Параллелизм: immutable данные не требуют синхронизации.

Как это работает

Python поддерживает элементы ФП: функции как объекты первого класса, map/filter/reduce, lambda, functools.partial, comprehensions. Чистые функции (без side effects) предсказуемы и тестируемы. functools.reduce последовательно применяет функцию к аккумулятору и элементам. pipe() — композиция через reduce.

16.1 Higher-order functions

Use cases

  • Декораторы: функция, принимающая функцию, возвращающая функцию.
  • Callback-обработчики: события, таймеры, коллбеки завершения.
  • Фабрики функций: создание специализированных функций на основе параметров.

Как это работает

Функции высшего порядка — принимают функцию как аргумент и/или возвращают функцию. map(func, iterable) применяет func к каждому элементу (лениво). filter(pred, iterable) — оставляет только элементы, где pred истинен. В Python чаще заменяют list comprehensions.

from typing import Callable, TypeVar

T = TypeVar("T")
U = TypeVar("U")

def map_list(func: Callable[[T], U], items: list[T]) -> list[U]:
    return [func(item) for item in items]

16.2 Pipeline / Compose

Use cases

  • Обработка данных: прочитать → распарсить → очистить → трансформировать → вывести.
  • Middleware в веб-фреймворках: цепочка обработчиков запроса.
  • ETL: Extract → Transform → Load.
from functools import reduce

def pipe(*functions: Callable) -> Callable:
    def inner(arg):
        return reduce(lambda x, f: f(x), functions, arg)
    return inner

process = pipe(str.strip, str.lower, lambda s: s.replace(" ", "_"))
result = process("  Hello World  ")

16.3 functools.partial

Use cases

  • Фиксация аргументов: int("FF", base=16) → hex = partial(int, base=16).
  • API-клиенты: fetch_user = partial(api.get, "/users") (с предустановленным URL).

Как это работает

partial(func, *args, **kwargs) — возвращает вызываемый объект, который запоминает часть аргументов. При вызове добавляет новые аргументы к уже сохранённым. partialmethod — то же для методов класса.

from functools import partial

def power(base: float, exp: float) -> float:
    return base ** exp

square = partial(power, exp=2)
print(square(5))  # 25

16.4 functools.reduce

Use cases

  • Объединение: конкатенация строк, сумма/произведение элементов.
  • Агрегация: найти максимум/минимум, построить словарь из списка.
  • Композиция: собрать пайплайн из функций.

Как это работает

reduce(func, iterable, [initial]) — последовательно применяет func к аккумулятору и каждому элементу. Без initial аккумулятор = первый элемент, итерация со второго. С initial — аккумулятор = initial, итерация с первого.

from functools import reduce
from operator import add, mul

print(reduce(add, [1, 2, 3, 4, 5]))  # 15
print(reduce(add, [1, 2, 3, 4, 5], 10))  # 25

16.5 Неизменяемость (Immutability)

Use cases

  • Value Objects в DDD: Money, DateRange, Coordinates — объект не меняется, операции возвращают новый.
  • Функциональные структуры: safe для многопоточности и кеширования.

Как это работает

frozen=True в dataclass запрещает присваивание атрибутов. Каждая операция (add, with_amount) возвращает новый экземпляр. Исходный объект не меняется. Это поверхностная immutability — если поле содержит список, его содержимое всё ещё можно менять.

from dataclasses import dataclass

@dataclass(frozen=True)
class Money:
    amount: float
    currency: str = "USD"

    def add(self, other: "Money") -> "Money":
        if self.currency != other.currency:
            raise ValueError("Currency mismatch")
        return Money(self.amount + other.amount, self.currency)

m1 = Money(100)
m2 = m1.add(Money(50))
print(m1)  # Money(100, 'USD') — не изменился
print(m2)  # Money(150, 'USD')

17. Dependency Injection

Use cases

  • Сервисы, зависящие от БД: UserService(repository: Repository).
  • Тестирование: подставить MockUserRepository вместо реального.
  • Смена реализации: EmailService → SmtpEmailService или MockEmailService.
  • Управление жизненным циклом: DI-контейнер создаёт синглтоны или новые экземпляры.

Как это работает

DI — паттерн, при котором зависимости внедряются в объект, а не создаются внутри. Это инверсия управления (IoC — Inversion of Control). Виды: Constructor Injection (через __init__), Setter Injection (через метод), Interface Injection. Python упрощает DI: отсутствие checked exceptions, динамическая типизация (или Protocol), возможность заменять атрибуты на лету.

17.1 Constructor Injection

Use cases

  • Сервисный слой: UserService не создаёт UserRepository сам, а получает извне.
  • Тесты: передать Mock() — легко.
  • Замена реализации: без изменения кода сервиса.

Как это работает

Все зависимости явно перечислены в __init__. Они сохраняются как атрибуты. При вызове методов используются эти зависимости. Это делает зависимости видимыми, тестируемыми и заменяемыми. Service Locator (антипаттерн) — альтернатива, где объект сам вызывает контейнер.

class UserService:
    def __init__(
        self,
        repository: Repository,
        logger: Logger,
        cache: Cache | None = None,
    ):
        self.repository = repository
        self.logger = logger
        self.cache = cache

    def get_user(self, user_id: str) -> User | None:
        if self.cache:
            cached = self.cache.get(user_id)
            if cached:
                self.logger.info(f"Cache hit for {user_id}")
                return cached

        user = self.repository.find_by_id(user_id)
        if user and self.cache:
            self.cache.set(user_id, user)

        return user

17.2 Простой DI-контейнер

Use cases

  • Автоматическое связывание: контейнер строит граф зависимостей.
  • Singleton/Transient: контролировать, когда создаётся объект.
  • Конфигурация: менять реализации в одном месте.
class Container:
    def __init__(self):
        self._services: dict[str, Any] = {}

    def register(self, name: str, service: Any) -> None:
        self._services[name] = service

    def resolve(self, name: str) -> Any:
        if name not in self._services:
            raise KeyError(f"Service {name} not registered")
        return self._services[name]

container = Container()
container.register("repository", UserRepository())
container.register("user_service", UserService(
    repository=container.resolve("repository"),
    logger=Logger(),
))

17.3 DI через протоколы

Use cases

  • Подмена реализации в тестах: MockEmailService реализует EmailService (протокол).
  • Слабая связанность: класс не импортирует конкретную реализацию.
class EmailService(Protocol):
    def send(self, to: str, subject: str, body: str) -> None: ...

class SmtpEmailService:
    def send(self, to: str, subject: str, body: str) -> None:
        print(f"Sending email to {to}...")

class MockEmailService:
    def send(self, to: str, subject: str, body: str) -> None:
        print(f"[MOCK] Would send email to {to}")

class NotificationService:
    def __init__(self, email_service: EmailService):
        self.email_service = email_service

    def notify(self, user: User, message: str) -> None:
        self.email_service.send(user.email, "Notification", message)

19.4 Фикстуры (fixtures)

Use cases

  • Подготовка БД: создать таблицы, заполнить тестовые данные.

  • Аутентификация: залогинить тестового пользователя.

Как это работает

Фикстура — функция с @pytest.fixture. Если тест имеет параметр с именем фикстуры, pytest вызывает фикстуру и передаёт результат. yield разделяет setup и teardown. Фикстуры строят граф зависимостей: если фикстура A зависит от B, то B выполнится первой и только один раз если одинаковый scope.

    Base.metadata.create_all(engine)
    session = Session(bind=engine)

    yield session

    session.close()

def test_user_repository(db_session):
    repo = UserRepository(db_session)
    user = repo.create(name="Alice", email="alice@example.com")
    assert user.id is not None

20. Продвинутое тестирование

Use cases

  • Изоляция внешних зависимостей: мок HTTP-запросов, БД, файловой системы.
  • Параллельный запуск: ускорить прогон тысяч тестов.
  • Property-based testing: найти редкие баги случайными входными данными.
  • Покрытие: убедиться, что весь код выполнен хотя бы раз.

Как это работает

Mock — объект, заменяющий реальную зависимость и записывающий вызовы. patch() временно заменяет объект в модуле моком. Coverage — инструмент, отслеживающий, какие строки кода выполняются. Property-based testing (hypothesis) генерирует случайные данные и проверяет свойства (инварианты).

20.1 Mocking с unittest.mock

Use cases

  • Внешний API: заменить HTTP-клиент, не делая реальных запросов.
  • Системное время: подменить datetime.now() на фиксированную дату.
  • Случайность: заменить random.random() на детерминированную последовательность.

Как это работает

Mock() — универсальный объект-заменитель. Любой вызов атрибута или метода возвращает новый Mock (без ошибки). return_value — что возвращает вызов. assert_called_once_with — проверка вызова с нужными аргументами. spec= ограничивает интерфейс существующим классом. side_effect — исключение или последовательность возвратов.

from unittest.mock import Mock, patch

def test_user_service_with_mock():

    service = UserService(mock_repo)
    user = service.get_user("1")

    assert user.name == "Alice"
    mock_repo.find_by_id.assert_called_once_with("1")

@patch("myapp.services.EmailService")
def test_send_notification(mock_email_service):
    service = NotificationService()
    service.send("user@example.com", "Hello")
    mock_email_service.send.assert_called_once()

20.8 Property-Based Testing (hypothesis)

Use cases

  • Поиск редких багов: миллионы случайных входов находят краевые случаи.
  • Математические свойства: коммутативность, ассоциативность, идемпотентность.
  • Проверка инвариантов: «после sort() список отсортирован».

Как это работает

hypothesis генерирует случайные данные согласно стратегиям (st.integers(), st.lists()). Декоратор @given запускает тест много раз с разными данными. При ошибке hypothesis находит минимальный воспроизводимый пример (shrinking). Стратегии можно комбинировать: st.dictionaries(st.text(), st.integers()).

from hypothesis import given, strategies as st

@given(st.integers(), st.integers())
def test_addition_commutative(a: int, b: int):
    assert a + b == b + a

@given(st.lists(st.integers()))
def test_sort_idempotent(lst: list[int]):
    sorted_once = sorted(lst)
    sorted_twice = sorted(sorted_once)
    assert sorted_once == sorted_twice

20.10 Лучшие практики тестирования
